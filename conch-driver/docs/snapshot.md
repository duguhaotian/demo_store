# agent安全沙箱内存快照方案设计

## 目的

基于内存块内容进行去重，实现多个microVM安全沙箱物理内存的极致复用。


## 关键工作流

### 快照制作阶段

1. 管理面组件conch，触发快照流程；
2. vmm 发起snapshot指令；
3. 导出microVM的diff内存页，写入本地缓存文件，生成内存diff快照文件；
4. 内存去重后台程序，根据内存diff快照文件的内存页，依次计算，去除已有的重复页并建立索引；
5. 构建去重后的内存diff快照文件元数据，并推送到快照服务器；

### 基于快照恢复阶段

1. 管理面组件conch，触发快照恢复流程；
2. conch准备基于快照对应的内存视图模板，如果本地没有缓存，则从快照服务器下载快照的内存视图模板（有元数据组成）；
    2.1 已有缓存，则跳过；
    2.2 下载快照的内存视图模板；
3. 调用vmm 发起restore指令，vmm restore内存新增恢复方式（当前方式uffd/mmap）：调用新增内核接口sys_restore(内存视图模板)；
4. vmm恢复运行，缺页为内存快照的RO内存页时：由新增内核机制，根据内存视图模板完成缺页处理，复用相同物理内存页；
